# 十一、对 Gradient 与 Backward 的理解

这是今天最重要的收获之一。

今天真正开始理解：

```text
loss.backward()
```

到底在做什么。

---

## 1. 梯度的本质

今天理解：

```text
梯度 = loss 下降方向
```

即：

```text
参数应该如何变化
才能让 loss 变小
```

数学形式：

```math
\frac{\partial Loss}{\partial W}
```

---

## 2. optimizer.step()

今天理解：

```python
optimizer.step()
```

本质是在：

```text
沿着 loss 下降方向
更新参数
```

数学形式：

```math
W_{new}=W_{old}-\eta\frac{\partial Loss}{\partial W}
```

---

## 3. Forward / Backward 本质

今天第一次真正理解：

```text
forward
```

不仅是在：

```text
计算结果
```

还会：

```text
记录 computation graph
```

而：

```text
backward
```

是在：

```text
沿着 graph 链式求导
```

---

# 十二、Inplace Operation 报错与 Autograd 理解

今天重点解决：

```text
one of the variables needed for gradient computation
has been modified by an inplace operation
```

并真正理解：

```text
PyTorch backward 需要 forward 时的旧 tensor
```

因此：

```python
x /= y
```

这种：

```text
原地修改（inplace）
```

会覆盖旧值。

最终使用：

```python
F.normalize(...)
```

解决问题。

今天第一次真正理解：

```text
Autograd Engine
```

的本质。
